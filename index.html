What’s the Difference Between Git and GitHub?

  • Git is the tool that will let you create a local repository (on your PC) and manage versions of your files
  • GitHub is an online service that will host your Git repositories (in the cloud)

Three environments

  1. Working directory (files)
  2. Staging (area) - is the middle ground between changes you have made to your local files and what you last committed (snapshot).
    git add… # To add working file(s) to staging
    git commit… # Captures a snapshot of the project's currently staged changes. Committed snapshots can be thought of as “safe” versions of a project—Git will never change them unless you explicitly ask it to.
  3. Remote GitHub Repository
    git push&hellip # To push staging repo to remote repo

What’s the Difference Between Fork and Clone?

  • FORK creates a copy from another user to your REMOTE repository
  • CLONE creates a copy from your REMOTE repository to your LOCAL repository

Git Bash commands

  • ls # List folders / files in the current directory
  • pwd # Provides the path of current directory
  • cd # Change directory
  • mkdir # Make a directory (folder)
  • touch # Make a file
  • clear # Clear the screen
  • status # Displays the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.
  • add # To add working file(s) to staging
  • commit # Captures a snapshot of the project's currently staged changes. Committed snapshots can be thought of as “safe” versions of a project—Git will never change them unless you explicitly ask it to.
  • push # To push staging repo UP to remote repo
  • pull # To pull remote repo DOWN to staging repo
  • rm -r NAME_OF_DIR # -r stands for recursive
